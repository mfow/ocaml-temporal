# Foundation and Deterministic Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Docker/Makefile repository foundation and a tested OCaml 5 runtime kernel whose typed workflows can schedule commands, suspend on futures, and resume deterministically from synthetic Temporal activations.

**Architecture:** Public modules expose typed codecs, workflow/activity definitions, structured errors, and futures while keeping effect constructors private. A small OCaml 5 deep effect handler owns deterministic fibers and one-shot continuations; a synthetic activation adapter proves the command/resolution boundary before the Rust Core FFI is introduced in the next phase.

**Tech Stack:** OCaml 5.2 compatibility floor, Dune 3.17+, Alcotest 1.8+, Docker Compose v2, Make, POSIX shell, Apache-2.0 project license.

## Global Constraints

- The public package is `temporal`; implementation-only libraries are private Dune libraries.
- Public expected failures use `result`; exceptions are caught only at the worker defect boundary.
- Effect constructors, continuations, mutable scheduler tables, and raw command IDs are private.
- Scheduler run order is FIFO by monotonically increasing fiber sequence.
- A future is resolved at most once and a captured continuation is resumed at most once.
- All public payload boundaries require an explicit `'a Temporal.Codec.t`.
- The OCaml compatibility floor is 5.2; CI also exercises current stable OCaml 5.5.
- Project source is Apache-2.0.
- Dependencies must satisfy the policy in `docs/implementation-roadmap.md`.
- Every task begins with a failing test, ends with green focused and aggregate tests, updates relevant documentation, and creates one Git commit.

- [x] **Step 3: Implement private suspension and deterministic queues**

The private promise retains its type parameters, so no `Obj.magic` is needed:

```ocaml
type ('a, 'e) promise
type _ Effect.t += Await : ('a, 'e) promise -> ('a, 'e) result Effect.t
```

`Future.t` wraps a private promise containing its scheduler identity, integer future ID, typed state, typed waiter list, and `outside_error : unit -> 'e`. `Future.await` first checks the promise state; ready values return directly and pending values perform `Await`. Calling outside the owning scheduler invokes `outside_error` and returns it through the future's error channel rather than leaking the effect. Activity and timer futures install `fun () -> Error.defect ~message:"future awaited outside its workflow scheduler"`.

`Scheduler` owns a deterministic runnable queue:

```ocaml
type runnable = Runnable : int * (unit -> unit) -> runnable
```

Each promise owns waiters of its own result type. Resolution converts each captured continuation into a closure placed on the scheduler's `Queue.t`; this preserves type safety without a heterogeneous value store. Assign sequence numbers at spawn and append resumed continuations in resolution-job order. The activation adapter, not hash-table iteration, determines resolution order. Each promise also registers one type-erased teardown closure with its scheduler so shutdown can discontinue every still-owned continuation.

`Future.both` resolves only after both inputs resolve and keeps left/right result positions independent of resolution order. If either side fails, return that error after observing both readiness states without canceling the sibling in this phase.

It rejects unknown or duplicate sequence resolutions as a structured bridge defect. It emits workflow completion exactly once. `Remove_from_cache` tears down all continuations without emitting a workflow command.
